# 变更说明

1.重构统一退出的实现方式。由于之前的统一退出是前端逐一退出所有登录的应用列表，该方案存在严重的缺陷，即稳定性不高，无法对于用户的行为进行控制。因此本方案进行了重构，借鉴CAS的实现方案，将用户的统一退出在服务端进行了实现。

2.增强了密钥的安全性。采用RSA非对称加密的方式获取密钥，用该密钥针对用户的解密。
